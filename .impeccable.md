# Design Context

## Product
0G Sandbox — a trusted, secure sandbox environment for vibe coding. Users get private, isolated containerized workspaces backed by TEE (hardware security enclave). The security and trustworthiness of the infrastructure is a core value proposition.

## Target Audience
Non-technical users. People who want to vibe code without worrying about setup, isolation, or security. They may not understand blockchain/TEE deeply but need to trust the product.

## Use Cases
1. Connect wallet → subscribe to a provider → deposit funds → create sandboxes
2. Manage running sandboxes (stop/start/delete/SSH)
3. Monitor balance and refund

## Brand Personality
- Web3 style — but trustworthy and secure, not hype-y or speculative
- Should feel: professional, safe, modern, slightly futuristic
- NOT: casino-like, overly flashy, dark-mode neon, or intimidating

## Platform
Desktop-first

## Current Pain Points
- Too plain / bland — lacks visual identity and brand presence
- Non-technical users may not understand the workflow (subscribe → deposit → create)
- No sense of security/trust communicated visually
